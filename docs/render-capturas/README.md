# Capturas necesarias para la entrega

Añade aquí las capturas que demuestran que la actividad funciona en Render.

Capturas recomendadas:

1. `01-health-render.png`  
   Captura de la URL pública de Render entrando a `/health`, por ejemplo:
   `https://TU-SERVICIO.onrender.com/health`

2. `02-logs-render.png`  
   Captura del panel de logs de Render mostrando entradas con formato JSON, por ejemplo:
   `"message":"HTTP request"`, `"message":"Health check passed"` o `"message":"Server started"`.

3. `03-render-healthcheck-config.png`  
   Captura de la configuración del servicio en Render donde se vea `Health Check Path: /health`.

4. `04-github-actions-ok.png`  
   Captura del workflow de GitHub Actions en verde, con tests, cobertura y despliegue.
